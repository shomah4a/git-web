# mode 表示を ls 形式に変換する

## セッション概要

### 実施内容

WorktreeView / RevisionTreeView のパーミッション表示を Git の 8 進数文字列（`100644`）から ls コマンド形式（`-rw-r--r--`）に変換した。

### 変更箇所

- `packages/front/src/format/entry.ts`: `formatMode` をビット厳密パース方式に書き換え
  - ファイルタイプビット → 先頭文字 (`d`, `l`, `m`, `-`)
  - 下位 9 ビット → owner/group/other の rwx
  - パース不能時は元の文字列をフォールバック返却
- `packages/front/src/format/entry.test.ts`: 新規テスト（8 ケース + formatSize 4 ケース）
- `packages/front/src/components/WorktreeView.vue`: `.col-mode` 幅を `5rem` → `7rem` に拡大
- `docs/adr/0034-mode-ls-format.md`: ADR 新規作成

### コミット

- `bde57d0` formatMode の ls 形式変換 (feature/mode-ls-format)
- `46c658a` 申し送りドキュメント (feature/mode-ls-format)
- `c968b60` worktree ディレクトリの mode を 040000 に固定 (main)

注: `c968b60` は誤って main に直接コミットされた。feature/mode-ls-format にも同内容の変更が必要だが、ユーザー判断でこのままとした。

### 設計判断

- `040000`（tree）は Git のビット値通り `d---------` と表示（ls の `drwxr-xr-x` とは異なる）
- `160000`（submodule）は `m---------`（POSIX に対応なし、`m` = module）
- setuid/setgid/sticky ビットは非対応（Git が返さないため）
