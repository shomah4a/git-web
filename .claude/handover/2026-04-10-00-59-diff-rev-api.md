# diff from/to セレクタ タスク A (API 側) 完了申し送り

## セッション概要

diff 画面の from / to 選択機能のうち、API 側 (タスク A) を実装した。
フロント側 (タスク B) は別セッションで対応予定。

対象ブランチ: `feature/diff-rev-api` (未マージ、未プッシュ)

## 主要成果

### 1. ADR 0018 起票

`docs/adr/0018-revision-extension-and-refs-api.md`

- ADR 0012 §5 の Revision 許可リストを BNF ベースの文法に差し替え
- `refs/heads/main`, `main`, `v1.0.0`, `HEAD^^`, `main~3`, `feature/foo^2` などを受理
- `-flag`, `HEAD;`, `HEAD@{N}`, 制御文字などは引き続き拒否
- `GET /api/refs` を仕様化 (q フィルタ / limit / head nullable / truncated フラグ)
- 二層防御として CLI 側で `git diff --end-of-options ...` を使う方針
- 実装後の指摘 M1 / M3 を反映し、取得コマンドの分離実行と unborn HEAD の挙動を訂正済み
- ADR 0012 / ADR 0009 の補遺にリンク追記

### 2. parseRevision 拡張

`packages/api/src/domain/revision.ts`

- BNF: `revision := sha | base modifier*` 形式
- `REFNAME_BODY_PATTERN` と `MODIFIER_SUFFIX_PATTERN` + 個別禁止条件 (`..` `//` 末尾 `/` 末尾 `.lock` `@{` 等)
- `InvalidRevisionError` に `reason: 'empty' | 'too-long' | 'forbidden-chars' | 'reflog-form' | 'bad-modifier' | 'shape'` を追加
- 既存の否定テスト (`main` / `v1.0.0` / `refs/heads/main`) は許可側に反転し、代表拒否ケースはシェルメタ混入に差し替え
- `diff-controller.test.ts`, `blob-controller.test.ts`, `blob-integration.test.ts`, `error-mapper.test.ts`, `errors.test.ts` も対応して更新

### 3. CLI 二層防御

`packages/api/src/adapter/git/cli-client.ts`

- `git diff --raw/--numstat/...` と `git diff -M ...` のすべてで `--end-of-options` を revision 引数の直前に配置
- git 2.24+ 前提 (ADR 記載)

### 4. /api/refs エンドポイント

- 新規ファイル:
  - `packages/api/src/domain/refs.ts` (ドメイン型 RefList / RefsQuery)
  - `packages/api/src/domain/refs-query.ts` (クエリパース)
  - `packages/api/src/domain/ports/git-refs-client.ts` (port)
  - `packages/api/src/service/refs-service.ts` (フィルタ + 切り詰め)
  - `packages/api/src/controller/refs-controller.ts`
  - `packages/common/src/refs.ts` (RefListDto)
- CliGitClient に `headRef` / `listBranches` / `listTags` を追加
- エラー型 `InvalidRefsQueryError` を追加、`error-mapper` で 400 `invalid_refs_query`
- `main.ts` に `{ method: 'GET', path: '/api/refs', handler: createRefsHandler(refsService) }` を登録

### 5. /api/refs の仕様サマリ

- クエリ: `q` (部分一致、省略時空文字列)、`limit` (1〜500、既定 100)
- 絞り込み: サーバ側 literal 部分一致 (大小無視)、正規表現不使用
- ブランチを先に詰め、残り枠でタグを詰める。超過時 `truncated: true`
- `head`: detached HEAD なら null、unborn HEAD なら git の仕様通り `main` が返る
