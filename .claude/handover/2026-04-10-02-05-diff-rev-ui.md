# diff from/to セレクタ UI タスク B 完了申し送り

## セッション概要

ADR 0018 (タスク A: API 側) の完了を受けて、フロントエンド側 (タスク B) を実装した。
diff 画面上部に from / to のリビジョンセレクタを追加し、ブランチ・タグ・HEAD 相対指定・SHA を選択または自由入力できるようにした。

対象ブランチ: `feature/diff-rev-ui` (未マージ、未プッシュ)

## 主要成果

### 1. ADR 0019 起票

`docs/adr/0019-diff-from-to-selector-ui.md`

- UI 構成: header に RevisionCombobox × 2 + 適用ボタン
- WORKTREE_SENTINEL = `(worktree)` を UI 仮想 ref として定義 (API クエリ上は undefined にマップ)
- RevisionCombobox の契約:
  - テキストフィールドは検索ボックスとして位置付け、入力中は親 modelValue を更新しない
  - 候補クリック / Enter は明示確定 → update:modelValue + submit emit
  - blur は暗黙確定 → update:modelValue のみ emit (自動適用なし)
  - debounce 200ms + lastIssuedGen 世代カウンタ
  - a11y 最小対応 (role=combobox, aria-expanded, role=option)
  - 検索ボックス視認性のため SVG inline の虫眼鏡アイコンを CSS background で配置
- DiffView の runDiffLoad シグネチャ拡張と range スナップショット化
- 自動適用: submit を DiffView が listen して runDiffLoad を自動発火
- ADR 0012 §5 / 0014 / 0017 の補遺に本 ADR へのリンクを追記

### 2. `packages/front/src/api/refs.ts` 新規

- `fetchRefs(q, limit)` を実装、レスポンスは型ガードで narrow (ADR 0010 準拠)
- URL は URLSearchParams で組み立て
- テスト 10 ケース

### 3. `packages/front/src/components/RevisionCombobox.vue` 新規

- 自前実装 (Vue 以外の新規依存なし)
- props: modelValue / initialRefs / allowWorktree / placeholder / hasError
- emits: update:modelValue / submit
- 内部: inputText / isOpen / highlightIdx / refs / lastIssuedGen / debounceHandle / blurHandle / isUnmounted
- commitExplicit (クリック / Enter) と commitImplicit (blur) を明確に分離
- debounce と blur の setTimeout は onBeforeUnmount で両方 clearTimeout する
- listboxId は Vue 3.5 の useId() を使用
- options 重複排除は Set で O(1) 判定
- テスト 21 ケース (fake timer 使用)

### 4. `packages/front/src/components/DiffView.vue` 改修

- `WORKTREE_SENTINEL = '(worktree)'` 定数追加
- `fromRev` / `toRev` / `initialRefs` state 追加
- `runDiffLoad(rangeArg?: DiffRangeQuery)` にシグネチャ拡張
- `currentRange()` / `resolveBlobRevs(range)` で range と blob rev を関数冒頭で 1 回だけスナップショット
- blob fetch の old/new rev を range から導出 (従来 'HEAD' / null ハードコード)
- `onMounted` で `fetchRefs('', 50)` を runDiffLoad と並列発火、結果を initialRefs へ
- header (`.rev-selector`) に RevisionCombobox × 2 と「適用」ボタンを追加
- 適用ボタンは `:disabled="loadingList"`
- `onRevSubmit` で combobox の submit を受けて自動適用
- `isUnmounted` フラグで onMounted 並列 fetchRefs の late emit を抑止
- テスト追加 9 ケース (run range / worktree / auto-apply / blur 非適用 / loading disable / has-error 伝播 等)

### 5. 開発プロセス

- 中間コミット (runDiffLoad シグネチャ拡張のみ、UI なし) を挟んで既存テスト 19 ケースがそのまま通過することを確認
- 11 ステップのコミット単位に細分化

## 残課題

### MEDIUM-2: invalid_revision の発生源特定 (from/to)

- 現状は listError を from/to 両方の combobox の hasError に流すため、from が正しいのに両方赤枠になる
- ADR 0019 §5 で容認した設計
- 本筋の解決は backend の error-mapper に `reason: { field: 'from' | 'to' }` を追加すること (ADR 0018 の再編が必要なため別タスク)
- Known Issue として本申し送りに記録する
