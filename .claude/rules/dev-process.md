# 開発プロセス

## ADRの記載

プランの最初に ADR を作ること。
ADR は docs/adr 以下に書き出すこと。

### 過去の判断事項の変更

過去に判断した事項が変更された場合、過去の判断を記録したADRは当時の判断のものとして残しつつ、変更があったADRへのリンクを追記しておくこと。

ADR 0005 での判断を ADR 0010 で直すときは以下のようにする:

- ADR 0010 を作る
- ADR 0005 のコンテキストに内容修正を行ったADR 0010 へのリンクを追加する


### 開発完了時

実装の最後に、開発途中の指摘などでADRとの齟齬が出てしまうようなケースをADRに反映すること。

## 申し送りドキュメント

申し送りドキュメントは .claude/handover 以下にmdとして記載する。

パスは `.claude/handover/YYYY-mm-dd-HH-MM-session-title.md` とする。

内容としては

- セッション概要
- TODO

として、次のセッションで前回セッションの概要ややり残しを把握するために用いる。

セッション開始時は最新のタイムスタンプの申し送りドキュメントを読み込むこと。

タスクが終わったら申し送りを書き込むこと。
