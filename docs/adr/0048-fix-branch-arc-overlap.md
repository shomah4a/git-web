# ADR 0048: グラフビューのノード重なり修正

## ステータス

承認済み

## コンテキスト

グラフビューで read-more により追加読み込みを行うと、本流と合流済みブランチのノードが重なって表示される不具合がある。

### 原因 1: 折りたたみ状態のブランチコミットがグラフに残存

`buildGraph` は全コミットをグラフノードとして追加していた。折りたたみ状態のブランチコミットは expand 疑似ノードの背後に隠れるべきだが、実際にはグラフ上にノードとエッジが残っていた。

これにより:
- ブランチコミットが BFS のルートとなり rank 0 からスタート（本流 HEAD と同じ rank）
- 本流ノードと同じ Y 座標に配置され、複数ノードが重なる
- read-more ノードが配列末尾のブランチコミットから接続され、本流の末端ではなくブランチの末端に配置される

### 原因 2: ブランチ配置が BFS ランクに依存

ブランチノードの Y 座標を BFS ランクベースで計算していたため、ブランチノードが本流ノードと同じ Y に配置され、read-more 後に重なりが顕在化していた。

## 決定

### 修正 1: 折りたたみブランチのコミット除外

`buildGraph` で本流 + 展開済みブランチのコミットのみをグラフに含めるようフィルタリングする。折りたたみ状態のブランチコミットはグラフから除外し、expand 疑似ノードのみを配置する。

read-more ノードは本流の最後のコミットから接続し、`isMainStream: true` として本流上に配置する。

### 修正 2: ブランチの楕円配置

BFS ランクベースの sin 弧配置を廃止し、ブランチノードを**楕円の円周上**に配置する方式に変更する。

- merge と fork が楕円の左端 (X=0、本流上) に位置する
- 楕円の中心を rx 分右にオフセットし、全ノードが X >= 0 に収まる
- merge から右回りに一周して fork に戻るループ形状
- 楕円の半径はノード数に応じて自動調整（ノード間最低弧長 240px を保証）
- expand-branch 疑似ノードは楕円ではなく親マージコミットの横下にオフセット配置

### 変更対象

- `packages/front/src/graph/build-graph.ts`: コミットフィルタリングと read-more 接続先の修正
- `packages/front/src/graph/use-graph-simulation.ts`: 楕円配置ロジック、expand ノード配置

### 変更しないもの

- `SimNode` 型定義
- `GraphView.vue` のテンプレートとイベントハンドラ

## 影響

- 折りたたみ状態ではブランチコミットが非表示になる（expand クリックで表示）
- read-more が本流末端に正しく配置される
- 展開済みブランチノードが楕円ループ上に配置され、本流と重ならない
