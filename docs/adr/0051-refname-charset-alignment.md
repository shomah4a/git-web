# 0051. refname 許可文字を git 仕様に合わせる

## ステータス

承認済み

## 文脈

ADR 0018 で定義した `refname` の許可文字パターンは `[A-Za-z0-9_./-]` であり、git の `check-ref-format` が許可する文字集合より狭い。ADR 0018 のデメリット欄にも「refname 文法はあえて git 公式の `check-ref-format` より絞っている」と記載されていたが、`+` を含むブランチ名が表示できない問題が報告され、仕様の未充足であることが判明した。

git の `check-ref-format` が**禁止**するのは以下のみである:

- ASCII 制御文字 (`\u0000`〜`\u001f` / `\u007f`)
- スペース、`~`、`^`、`:`、`?`、`*`、`[`、`\`
- `..` (連続ドット)
- `@{` (reflog 記法)
- 先頭 `-`
- 末尾 `.lock`
- 末尾 `/`
- 連続スラッシュ `//`

上記に該当しない文字はすべて合法である。現在の `REFNAME_BODY_PATTERN` が不当に拒否している文字は以下の 7 つ:

`+`、`@`、`!`、`#`、`%`、`,`、`=`

なお、シェルメタ文字（`; & | \` $ ( ) < > " ' { }`等）は`containsForbiddenChars` で defense-in-depth として別途禁止されており、`REFNAME_BODY_PATTERN` の変更による影響はない。

## 決定

### 1. `REFNAME_BODY_PATTERN` の拡張

許可文字に `+ @ ! # % , =` を追加する。

変更前:

```
refname := [A-Za-z0-9_.] [A-Za-z0-9_./-]{0,254}
```

変更後:

```
refname := [A-Za-z0-9_.+@!#%,=] [A-Za-z0-9_.+@!#%,=/-]{0,254}
```

### 2. 単独 `@` の扱い

git の仕様で単独の `@` は `HEAD` のエイリアスであり ref 名としては存在しないが、リビジョン指定としては `HEAD` と同義で合法である。`parseRevision` はリビジョン指定のバリデーションであるため、単独 `@` を拒否しない。

### 3. 既存の防御層との関係

- `containsForbiddenChars`: 変更なし。シェルメタ文字の defense-in-depth はそのまま維持
- `splitBaseAndModifier`: 変更なし。`^` `~` による分割ロジックは追加文字と無関係
- CLI 側 `--end-of-options`: 変更なし

## 結果

### メリット

- git の `check-ref-format` で合法な ref 名を正しく受理できるようになる
- `+` を含むブランチ名（dependabot やセマンティックバージョンのプレリリースタグ等）が表示可能になる

### デメリット

- 許可文字の拡大により、URL エンコーディングが必要な文字が増える。ただしフロントエンドは `URLSearchParams` を使用しており、`+` → `%2B` の事前置換も実装済みのため追加対応は不要
