# ADR 0050: diff view のコンテキスト展開機能

## ステータス

承認

## コンテキスト

現在の diff view は `git diff` が出力する unified diff のコンテキスト行（デフォルト 3 行）のみを表示している。GitHub の diff view には hunk 間のギャップを展開して前後のコードを確認できるボタンがあり、変更箇所の文脈を理解するのに有用である。

blob データは既にハイライト用に取得・トークン化済みで、`tokenMap` にファイル全体分の `HighlightedLines` として保持されている（ADR 0017）。追加の API 呼び出しなしに、フロントエンドのみで展開機能を実現できる。

## 決定

### ギャップの定義

hunk 間の未表示領域を「ギャップ」と呼ぶ。ギャップは以下の 3 種類がある。

- **先頭ギャップ**: ファイル先頭 ~ 最初の hunk の開始行の前
- **hunk 間ギャップ**: hunk i の終了行の次 ~ hunk i+1 の開始行の前
- **末尾ギャップ**: 最後の hunk の終了行の次 ~ ファイル末尾

old 側と new 側で行数が異なる場合があるため、ギャップの行範囲は old/new で独立して管理する。

### 展開の仕組み

各ギャップに対して「展開済み行数」を reactive state として管理する。

```typescript
type GapExpansion = {
  expandedDown: number // ギャップ上端から下に展開した行数
  expandedUp: number // ギャップ下端から上に展開した行数
}
```

- 展開ボタンのクリックで 10 行ずつ追加表示する
- `expandedDown + expandedUp >= ギャップ行数` の場合は全行を表示する（重複排除）
- ギャップが完全に埋まったら展開ボタンを非表示にする

### 展開ボタンの配置

展開ボタンは各 hunk に紐づけて配置する。

- hunk 上端に `↑` ボタン: 「この hunk の上にコンテキストを追加する」操作
- hunk 下端に `↓` ボタン: 「この hunk の下にコンテキストを追加する」操作

ボタンは常に展開済み行の外側（展開の先端）に位置する。クリックするたびにボタンが先端方向に移動し、展開済み行は hunk とボタンの間に挿入される。

```
[↑]              ← 展開の先端
[up 展開済み行]
@@ hunk @@
hunk の行
[down 展開済み行]
[↓]              ← 展開の先端
```

### hunk 結合

ギャップが完全に展開されて埋まった場合、後続 hunk の `@@ ... @@` ヘッダーを非表示にし、展開行と hunk の行が視覚的にシームレスにつながるようにする。

### 展開行の描画

- context 行と同じスタイル（背景透明、行番号あり）で描画する
- tokenMap のトークンが利用可能ならシンタックスハイライト付き
- 展開ブロックは既存の `.hunk` と同じ flex レイアウト（左右分割）で描画する

### ロジックの分離

ギャップ計算と展開行算出のロジックは `packages/front/src/diff/expand-context.ts` に純粋関数として切り出す。`pair-lines.ts` と同じパターン。

### 制約

- `tokenMap` にデータがないファイル（binary、blob 取得失敗、大容量）では展開ボタンを表示しない
- revision 切り替え時に展開 state をリセットする
