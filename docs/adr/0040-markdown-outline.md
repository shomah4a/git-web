# 0040. Markdown レンダリングビューのアウトライン表示

## ステータス

承認済み (2026-04-15)

## コンテキスト

Markdown ファイルのレンダリングビューで、ドキュメントの構造を把握しにくい。
見出し一覧をアウトラインとして表示することで、ドキュメント内のナビゲーションを改善したい。

印刷ビュー (ADR 0039 の chromeless モード) ではアウトラインは不要である。

## 決定

### DOM ベースの見出し抽出

`renderMarkdown` の戻り値型は変更せず、レンダリング済み HTML が DOM に反映された後に `h1`-`h6` 要素を動的に取得する。

理由:

- render.ts や BlobContentState の型変更が不要であり、既存コードへの波及がない
- レンダラに依存しないため、将来別のレンダラ (例: AsciiDoc, reStructuredText) が追加された場合も `v-html` で出力される限り自動的に対応できる

### 見出しへの id 付与

DOM から見出しを収集する際、`id` 属性が未設定の見出しには動的に `id` を付与する。
GitHub 方式に倣い、テキストをスラッグ化（小文字化、スペースを `-` に、記号除去）して `id` とする。
同名の見出しが複数存在する場合は連番サフィックス (`-1`, `-2`, ...) を付与する。

### レンダリング同期

`v-html` のソースデータ (`state.renderedMarkdown`) を `watch` し、`nextTick` で DOM 更新を待ってから見出しを再収集する。

### 表示位置

コンテンツ本体の右側にサイドバーとして表示する。
見出しのレベルに応じてインデントし、クリックで該当箇所にスクロールする。

### 非表示条件

- `chromeless` prop が true の場合（印刷ビュー）
- `@media print`

### DOMPurify の前提

DOMPurify のデフォルト設定では `id` 属性は許可されている。
ただし `SANITIZE_NAMED_PROPS` オプションを有効にすると `id` にプレフィックスが付与される。
現状このオプションは未使用であり、本機能はその前提で動作する。

## 実装方針

- 見出し抽出ロジックは composable (`use-outline.ts`) として分離する
- BlobContent.vue にアウトラインサイドバーを追加する
