# ADR 0046: コミット履歴ビュー

## ステータス

承認

## コンテキスト

git-web はリポジトリの tree / blob / diff を閲覧できるが、コミット履歴を一覧する機能がない。ブランチやタグの指すコミットから過去の変更をたどるには、ターミナルで `git log` を実行する必要がある。

## 決定

### ルーティング

- フロント: `/commits` ルートを新設する
- API: `GET /api/commits` エンドポイントを新設する
- App.vue のタブナビゲーションに History タブを追加する

### クエリパラメータ

| パラメータ | 型              | デフォルト | 説明                                        |
| ---------- | --------------- | ---------- | ------------------------------------------- |
| `rev`      | Revision        | `HEAD`     | 起点リビジョン (ブランチ名、タグ名、SHA 等) |
| `after`    | SHA (4-40桁hex) | なし       | カーソル: このコミットの次から取得          |
| `limit`    | number          | `20`       | 1 ページあたりの取得件数                    |
| `path`     | string          | なし       | パス絞り込み (`git log -- <path>`)          |

### リビジョン選択

既存の `RevisionCombobox` コンポーネントを再利用する。`allowWorktree: false` とし、worktree sentinel は選択肢に含めない。適用ボタンを RevisionTreeView と同パターンで設置する。

### ページネーション

カーソルベース方式を採用する。

- `limit + 1` 件を git に要求し、超過分があれば `hasMore: true` を返す
- フロントは「もっと読み込む」ボタンを表示し、末尾コミットの SHA を `after` パラメータとして次のリクエストを送る
- `after` パラメータは SHA 形式 (`/^[0-9a-f]{4,40}$/`) のみ許可する (カーソル用途のため)
- `after` 指定時は `after~1` を単独起点として git log に渡す。rev との union にしない

### git log フォーマット

```
git log --format=%x00%H%x01%P%x01%an%x01%ae%x01%at%x01%s%x01%b%x01 --numstat
```

- `%x00` (NUL) をレコードセパレータ、`%x01` (SOH) をフィールドセパレータとする
- これによりコミットメッセージに改行が含まれていても安全にパースできる
- `%P` で親コミットのハッシュ一覧を取得し、親の数からマージコミットを判定する
- `%at` で UNIX epoch 秒を取得する (タイムゾーン独立)
- `--numstat` は各コミットの後に改行区切りで出力される
- バイナリファイルの numstat は `-` `-` になるため、パーサーで数値変換失敗時は 0 として扱う
- マージコミットの numstat は空になりうる (diff が出ない場合)

### 日時の扱い

コーディング規約 (070) に従い、タイムゾーン独立な値を使う。

- git log: `%at` (UNIX epoch 秒) を使用。`%aI` (ISO 8601 + offset) は使わない
- API/DTO: `date: number` として UNIX epoch 秒を渡す
- フロント: `new Date(epoch * 1000)` でブラウザのローカルタイムゾーンに変換し、タイムゾーンオフセットも併せて表示する

### マージコミット判定

- `%P` (親ハッシュ一覧) のスペース区切りの数で判定する
- `parentCount >= 2` の場合にマージコミットとみなす
- フロントでは Type カラムに merge バッジを表示する

### セキュリティ

- `rev` パラメータは既存の `parseRevision` でバリデーションする
- `after` パラメータは SHA 専用のバリデーション関数を新設する
- git コマンド実行時は `--end-of-options` を rev の前に置く (既存の二層防御パターン)
- パス絞り込みは `-- <path>` で rev と分離する

### コミットからの遷移先

各コミット行に以下の導線を設ける:

- diff ビュー: `/diff?from=<sha>^&to=<sha>` — 当該コミットの変更差分 (マージコミットでは first-parent diff)
- revision tree: `/tree?rev=<sha>` — 当該コミット時点のツリー

### 表示項目

- コミット SHA (短縮形)
- Type (マージコミットの場合 merge バッジ)
- コミットメッセージ (subject、body は省略表示)
- 著者名 (メールアドレスは title 属性)
- 日時 (ブラウザのローカルタイムゾーン + オフセット)
- 差分統計: ファイル数、insertions (+)、deletions (-)
- diff / tree へのリンク

### テーブル UI

- テーブルヘッダは `position: sticky` で `--header-height` の直下に固定する

## 非採用案

### offset ベースのページネーション (`--skip N`)

`git log --skip` は内部的にスキップ分のコミットを走査するため、深いページでは O(N) の計算量がかかる。カーソルベースなら `<sha>` 以降のログを直接取得でき、深いページでも一定のコスト。

### 専用の `/commit/:sha` 詳細ビュー

既存の `/diff` ビューで `from=<sha>^&to=<sha>` とすれば同等の表示が得られるため、初期スコープでは専用ビューを作らない。

### ISO 8601 形式の日時 (`%aI`)

`%aI` はタイムゾーンオフセット付きの文字列であり、タイムゾーン依存型に該当する。コーディング規約 (070) の「サービス層やモデル層ではタイムゾーンから独立した型を用いて処理する」に反するため、UNIX epoch 秒 (`%at`) を採用した。
